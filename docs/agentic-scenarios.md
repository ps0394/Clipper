# Proposed Agentic Scenario Set

## Scenarios

| Scenario | What is it? | What gate is tested? | What we will observe (signals) | Canonical Prompts | Test setup |
|----------|-------------|---------------------|-------------------------------|-------------------|------------|
| **0) Agent content retrievability baseline** Impact: high | Measure whether HTTP crawlers and on-demand retrieval agents can access, extract, and use Learn content from the live website. Applies to search index crawlers, Copilot training pipelines, and Copilot retrieval-augmented agents that pull Learn content at query time. | Access | Content extraction quality for crawlers building search/training indexes and retrieval agents grounding responses on demand. Signals: Schema.org structured data completeness (can crawlers classify content?), Mozilla Readability content extraction ratio (can agents isolate article content from chrome?), Dublin Core / OpenGraph metadata presence (things like date, product, etc.), WCAG/axe-core DOM traversability, robots.txt crawl permissions and cache header support. Directly impacts index quality for scenarios 1-5 and grounding fidelity for on-demand retrieval. | Standards-based evaluation across 6 pillars (W3C Semantic HTML, Content Extractability, Schema.org Structured Data, DOM Navigability, Metadata Completeness, HTTP Compliance) against representative URLs per ms.topic type. Competitive benchmark against Google Cloud, AWS, MDN, Wikipedia, Stripe, Mintlify. Current baseline: Learn 60/100, competitive avg 51/100. | Automated evaluation against 16 Learn URLs (12 ms.topic types + Q&A) + 6 competitive/exemplar sites using industry-standard tools (axe-core, extruct, Mozilla Readability, HTTP header analysis). Run periodically to track score changes as template fixes ship. No agent endpoint or MCP required — evaluates the content surface that crawlers and retrieval agents consume. |
| **1) Deployment / configuration next steps** Impact: very high | Recommended next steps for deploying and configuring Azure resources | Success | Directly supports agent guided execution in-product and orchestration. This is where agents move from architecture discovery to deployment and config in real environments inside portal and product blades. | Customer prompt: 'set up Key Vault and private endpoint and firewall rules for this app' inside Azure Portal, with the agent sequencing steps | Examples: endpoint (GitHub Copilot Chat, Copilot CLI, Claude Code, Azure Copilot, etc.), starting state (what project/workload/env are we using so we're not comparing apples to oranges), tools enabled (ex: web search only, Learn MCP on/off, Azure MCP on/off) because MCP is not default |
| **2) MCP-mediated task completion** Impact: very high | MCP retrieved Learn content in coding/scripting and mediated execution flows | Execution | Aligns with coding and script agents in tooling environments (CLI, IDE, etc.) and measurable through MCP and ACE. In mediated partner workflows Learn can act as the skill generation layer that agents call into to apply authoritative guidance across execution tasks. | In VS Code, the agent pulls Learn guidance via MCP and produces Terraform for managed identity and deployment steps | |
| **3) Well-Architected assessment** Impact: high | Assess whether a workload is well architected | Execution → Success | Supports architecture decisions prior to deployment by enabling agents to evaluate patterns and tradeoffs using authoritative guidance (WAF/CAF/AAC) and the solutions recommender. | Customer prompt: 'is this AKS setup well-architected for reliability and cost?' Agent flags missing autoscaling and recommends zone redundancy | |
| **4) Open-ended discovery (AEO)** Impact: high | Surface authoritative guidance for open ended queries | Access | Supports partner owned architecture discovery where agents must decide what guidance to retrieve before recommending execution paths (deployment, architecture, etc.) | Customer prompt: 'what is the recommended approach for hybrid identity across AWS and Azure?' Agent pulls the right identity and migration guidance | |
| **5) Authority grounded architectural reasoning** Impact: high / medium high | Answer architectural 'why' questions | Execution | Builds trust and justification layer needed for adoption of partner owned recommendations through grounded source of truth. | Customer prompt: 'why is Front Door vs Traffic Manager here?' Agent explains tradeoffs and conditions, grounded in Microsoft guidance. | |
