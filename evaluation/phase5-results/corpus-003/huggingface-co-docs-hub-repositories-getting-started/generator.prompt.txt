Getting Started with Repositories

This beginner-friendly guide will help you get the basic skills you need to create and manage your repository on the Hub. Each section builds on the previous one, so feel free to choose where to start!

Requirements

This document shows how to handle repositories through the web interface as well as through the terminal. There are no requirements if working with the UI. If you want to work with the terminal, please follow these installation instructions.

If you do not have 
git
 available as a CLI command yet, you will need to 
install Git
 for your platform. You will also need to 
install Git-Xet
, which will be used to handle large files such as images and model weights.

To be able to download and upload large files from Git, you need to install the 
Git Xet
 extension.

To be able to push your code to the Hub, you’ll need to authenticate somehow. The easiest way to do this is by installing the 
hf
 CLI
 and running the login command:

The content in the Getting Started section of this document is also available as a video!

Creating a repository

Using the Hub’s web interface you can easily create repositories, add files (even large ones!), explore models, visualize diffs, and much more. There are three kinds of repositories on the Hub, and in this guide you’ll be creating a

Adding files to a repository (Web UI)

To add files to your repository via the web UI, start by selecting the

Creating a new file

Choosing to create a new file will take you to the following editor screen, where you can choose a name for your file, add content, and save your file with a message that summarizes your changes. Instead of directly committing the new file to your repo’s

Uploading a file

If you choose

As with creating new files, you can select

Adding files to a repository (CLI)

You can upload files to your repository directly from the terminal using the 
hf
 CLI

hf upload your-username/your-model-name ./my-model-directory

The 
hf
