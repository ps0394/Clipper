You are generating question/answer pairs for a calibration corpus that measures how well a documentation page supports retrieval-based question answering.

The document below is the full extracted text of a single web page. Your job is to produce 5 fact-based question/answer pairs whose answers are unambiguously stated in the document. These Q/A pairs will later be used to test whether other LLMs can answer the same questions using only the document text.

Strict rules:

1. Every answer must be directly and unambiguously supported by sentences in the document. If the answer requires outside knowledge, common sense, or inference beyond what the document states, do not include it.
2. Questions must be self-contained. Do not write questions that only make sense if you have already read the document (e.g., "What is the second step?"). Questions should make sense to a reader who has not seen the document.
3. Questions must span different sections of the document, not cluster in one easy region.
4. Questions must be fact-based. No opinion questions, no "why should you use X" questions unless the document itself states the reason.
5. For each Q/A pair, list the supporting sentences verbatim from the document.

Output format: a single JSON array with exactly 5 objects, each with keys `question`, `answer`, and `supporting_sentences` (an array of strings). No prose before or after. Do not wrap the JSON in a code block.

Vercel gives you multiple ways to interact with and configure your Vercel Projects. With the command-line interface (CLI) you can interact with the Vercel platform using a terminal, or through an automated system, enabling you to 
retrieve logs
, manage 
certificates
, replicate your deployment environment 
locally
, manage Domain Name System (DNS) 
records
, and more.

If you'd like to interface with the platform programmatically, check out the 
REST API documentation
.

To download and install Vercel CLI, run the following command:

pnpm
bun
yarn
npm

When there is a new release of Vercel CLI, running any command will show you a message letting you know that an update is available.

If you have installed our command-line interface through 
npm
 or 
Yarn
, the easiest way to update it is by running the installation command yet again.

pnpm
bun
yarn
npm
yarn global add vercel@latest

If you see permission errors, please read npm's 
official guide
. Yarn depends on the same configuration as npm.

The 
--version
 option can be used to verify the version of Vercel CLI currently being used.

Using the 
vercel
 command with the 
--version
 option.

Vercel CLI requires you to log in and authenticate before accessing resources or performing administrative tasks. In a terminal environment, you can use 
vercel login
, which requires manual input. In a CI/CD environment where manual input is not possible, you can create a token on your 
tokens page
 and then authenticate using one of these methods:

Set the 
VERCEL_TOKEN
 environment variable

Pass the 
--token
 option
 to the command

Using the 
VERCEL_TOKEN
 environment variable is recommended for CI/CD because it avoids exposing the token in command-line arguments, which can be visible in process lists and logs. If both are provided, the 
--token
 flag takes precedence over the environment variable.
