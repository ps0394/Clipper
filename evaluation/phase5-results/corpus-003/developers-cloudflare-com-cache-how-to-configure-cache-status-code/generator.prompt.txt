TTL handling for 304 and 200 status codes

If a TTL is not explicitly set for status code 
304

200

If a user explicitly sets a different TTL for

When a

Once the asset expires and we revalidate with the origin, if the origin returns a

For example, if a user specifies a TTL of one hour for status

This behavior is likely undesirable unless the user has a specific use case. Therefore, users should ensure that the TTL for

200
