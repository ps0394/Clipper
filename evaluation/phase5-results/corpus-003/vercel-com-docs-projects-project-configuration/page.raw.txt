Vercel automatically detects your framework and sets sensible defaults for builds, deployments, and routing. Project configuration lets you override these defaults to control builds, routing rules, function behavior, scheduled tasks, image optimization, and more.

In addition to configuring your project through 
Project Settings
, you have the following options:

Static file-based configuration
 - Static JSON configuration in your repository

Programmatic file-based configuration
 - Dynamic TypeScript configuration that runs at build time

Global CLI configuration
 - System-wide Vercel CLI settings

Each method lets you control different aspects of your project.

File-based configuration lives in your repository and gets version-controlled with your code. You can use either 
vercel.json
 for static configuration or 
vercel.ts
 for programmatic configuration that runs at build time. Both support the same properties, but 
vercel.ts
 lets you generate configuration dynamically using environment variables, API calls, or other build-time logic. You can only use one configuration file per project.

Global Configuration
 affects how Vercel CLI behaves on your machine. These settings are stored in your user directory and apply across all projects.
