In this tutorial, you prepare a multi-container application to use in Kubernetes.

You use existing development tools like Docker Compose to locally build and test the application.

Clone a sample application source from GitHub.

Create a container image from the sample application source.

Test the multi-container application in a local Docker environment.

Once completed, the following application runs in your local development environment:

In later tutorials, you upload the container image to an Azure Container Registry (ACR), and then deploy it into an AKS cluster.

To complete this tutorial, you need a local Docker development environment running Linux containers.

Azure Cloud Shell doesn't include the Docker components required to complete every step in these tutorials.

Therefore, we recommend using a full Docker development environment.

Create a directory on your computer and switch to that directory in your terminal session, like Bash.

git clone https://github.com/Azure-Samples/aks-store-demo.git

Change into the cloned directory.

Change into the new directory to load all the files from the azd template.

azd init --template aks-store-demo

image: rabbitmq:3.13.2-management-alpine

- "RABBITMQ_DEFAULT_USER=username"

- "RABBITMQ_DEFAULT_PASS=password"

test: ["CMD", "rabbitmqctl", "status"]

- ./rabbitmq_enabled_plugins:/etc/rabbitmq/enabled_plugins

container_name: 'order-service'

test: ["CMD", "wget", "-O", "/dev/null", "-q", "http://order-service:3000/health"]

container_name: 'product-service'

test: ["CMD", "wget", "-O", "/dev/null", "-q", "http://product-service:3002/health"]

- AI_SERVICE_URL=http://ai-service:5001/

test: ["CMD", "wget", "-O", "/dev/null", "-q", "http://store-front:80/health"]

- VUE_APP_PRODUCT_SERVICE_URL=http://product-service:3002/

- VUE_APP_ORDER_SERVICE_URL=http://order-service:3000/

Create container images and run application

docker compose -f docker-compose-quickstart.yml up -d
